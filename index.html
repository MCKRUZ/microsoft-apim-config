MCKRUZ · AI Governance · Golden Copy v1.0

APIM as the AI Gatekeeper The single chokepoint in front of every model call, tool invocation, and agent-to-agent hand-off.

The trust boundary moved. Model-level safety governs the words a model produces; it cannot govern what an agent does. Who may act, how much they may spend, which systems they reach, and a record of all of it — those live at the one point every action crosses: the API gateway.

The invariant: every model, tool, and agent call transits the gateway — and with network isolation on, it's enforced by the network, not by an honor system.
View the repo → Enterprise architecture →

In 30 seconds

When agents can act, the controls that matter aren't in the model — they're at the gateway: spend caps, content safety, identity, and an audit record.

This is a deployable reference — Bicep IaC + deep docs, not slideware. One control plane (Azure API Management) governs three traffic surfaces. Every capability is a feature flag: dev reproduces a simple seed; regulated turns nearly everything on. Built to survive a Dell/T-Mobile-class org. Clone it, azd up, and the gateway is governed.

One control plane · three traffic surfaces

Surface 1

agent → model GA

Azure OpenAI

Token spend caps (rate + monthly quota with pre-flight estimation), content safety (Prompt Shields), semantic cache, and per-team cost attribution — one ordered policy.

Surface 2

agent → tool Preview

MCP servers

Rate-limit, identity, and an agent-id audit trace on tool traffic. Whole-server scope; no response-body access (streaming-safe).

Surface 3

agent → agent Preview

A2A JSON-RPC

Rate-limit and identity on agent-to-agent hand-offs, with OpenTelemetry gen_ai.agent.id attribution as the audit record.

The four GA controls (composed from flags)

llm-token-limit

Spend cap

Per-minute TPM + monthly quota, keyed per team. Pre-flight estimation rejects over-budget calls before they bill the model.

llm-emit-token-metric

Cost attribution

Per-team / per-agent token metrics into Application Insights — the FinOps view.

llm-content-safety

Content safety

Prompt Shields block jailbreaks and indirect injection before the model — request and, optionally, response.

llm-semantic-cache

Semantic cache

Returns cached completions for semantically similar prompts (Azure Managed Redis + RediSearch) — tokens not re-billed.

Everything is a toggle

One profile, every capability a flag

A profile selects a flag set; any flag is overridable per deploy. Flags gate conditional module deployment and compose the policy itself — so a disabled control leaves no dead policy behind. The per-flag wiring audit (Bicep-gated · policy-composed · informational · declared-future) is honest about every one.

# pick a posture
azd env set GOV_PROFILE prod        # dev | test | prod | regulated
azd up

# or override one flag on any profile
az deployment sub create -l eastus2 -f infra/main.bicep \
  -p infra/main.parameters.json -p profile=dev \
  -p flagOverrides='{"networkIsolation":true}'
